LLM 동작 정책서

Sprouting Engine v3.0 · LLM이 언제 / 어떻게 / 얼마나 호출되는가

제1조 (목적)

본 정책서는 본 시스템에서 LLM(대규모 언어 모델)이 호출되는 모든 지점의 시점·형식·제약·폴백을 단일 진실 출처(Single Source of Truth)로 정리한다. LLM은 비결정적이고 비용·지연이 발생하는 자원이므로, 호출은 학술적·운영적으로 정당화된 지점에 한정된다.

본 정책서는 헌법(주장글쓰기 규칙)과 동급의 운영 문서이며, 충돌 시 본 정책서가 시스템 행동의 우선 기준이 된다.

정책 6대 원칙
  1. 최소 호출 원칙 — 결정론으로 가능한 측정은 결정론으로 처리하고, LLM은 "판단(judgment)"이 필요한 지점에만 사용한다.
  2. 가드레일 우선 — Question Designer는 출력이 의문문이 아니면 차단·재생성한다 (논문 §2.3).
  3. 독립 컨텍스트 — Appeal Judge는 직전 비판을 모르는 별도 컨텍스트에서 호출하여 자기참조 편향을 회피한다.
  4. 스키마 강제 — 모든 LLM 호출은 구조화 출력(JSON)을 요구하고, 스키마 위반 시 폴백 후 1회 재시도.
  5. 지연 예산 — 한 피드백 사이클의 LLM 누적 시간은 5초를 넘지 않는다.
  6. 학술적 정당성 — 모든 호출 지점은 ICCE2026 논문의 인용에 1:1 매핑된다.

제2조 (LLM 호출이 허용된 지점 — 5개 도구별 정책)

본 시스템은 논문 §7이 명기한 5개 도구만 운영한다. 각 도구의 LLM 사용 여부는 다음과 같다.

# 도구 LLM 사용 호출 빈도 비고
T1 S_seed Estimation 부분 (1/4 신호) 블록 평가당 1회 4신호 중 complexity만 LLM, 나머지 3신호(structure, self_correct, uncertainty)는 결정론 측정
T2 Question Designer 🔒 LLM 필수 학생이 [💬 소크라테스에게 물어보기] 클릭 시 1회 출력은 반드시 의문문(?)으로 종료. 가드레일 위반 시 1회 재생성, 그래도 위반 시 정적 풀에서 폴백.
T3 Aporia Detection 결정론 (LLM 미사용) 매 학생 입력 후 트리거 평가 같은 주장 3회 반복 + 응답시간 2σ + "모르겠다/어렵다" 마커 — 3개 중 2개 충족
T4 Emotional Monitoring 결정론 (LLM 미사용) 매 학생 입력 후 트리거 평가 타이머 임박 + 글자 수 정체 + 부정 정서 마커 + 항소 연속 클릭 — 휴리스틱 결합 (Eisenberg, 2010)
T5 Appeal Arbitration LLM 필수 (독립 컨텍스트) 학생 [항소하기] 클릭 시 1회 이전 비판 컨텍스트 비공개. rubric 강제: new_evidence / logical_flaw / perspective_shift 중 1개에 매핑.
✅ 5개 도구 합산 — 한 블록 평가 사이클당 LLM 호출은 최대 2회 (S_seed complexity 1회 + Question Designer 1회). Appeal은 별도 사용자 액션 시 +1회.
❌ 금지: 위 5개 도구 외 어떤 LLM 호출도 허용되지 않는다. "정답 검토", "글 자동 완성", "맞춤법 교정", "추천 주제 생성" 등은 본 정책서가 명시 차단.

제3조 (호출 시점 — 사용자 액션 ↔ LLM 호출 시퀀스)

학생/교사의 액션과 LLM 호출의 매핑은 다음과 같다. 명시되지 않은 액션은 LLM을 호출하지 않는다.

사용자 액션 LLM 호출 설명
학생: 텍스트 타이핑 (실시간) 호출 없음 입력 중에는 LLM 미호출. 결정론 신호만 갱신.
학생: [💬 소크라테스에게 물어보기] T1 + T2 현재 블록 텍스트로 S_seed 측정 → Question Designer 호출. 잔여 횟수 1 차감.
학생: [➡️ 다음 블록으로] 호출 없음 블록 전이는 상태 변화만. 다음 블록에서 첫 [💬] 시 호출.
학생: [🎯 다 썼어! → 최종 평가] T1 × N + T2 × 1 (총평) 모든 블록 S_seed 일괄 + 종합 코멘트 생성. (※ Phase 1 한정 평가 — 5축 평가 LLM 사용은 본 프로토타입 범위 외)
학생: [⚖️ 항소하기] T5 독립 컨텍스트에서 appeal_judge 1회 호출.
학생: [🗺️ 도와줘! → 구조도] 호출 없음 정적 모달. 구조도 카드는 사전 작성된 정적 콘텐츠.
시스템: Aporia / Distress 감지 호출 없음 결정론 트리거. 임계 충족 시 모드 전환 플래그 설정.
교사: TC-02 모니터링 진입 호출 없음 누적된 S_seed 데이터를 시각화만. AI 자동 코멘트 생성은 P1.

// 학생이 [💬 소크라테스에게 물어보기]를 누른 시점부터 발화 노출까지

학생 클릭
   │
   ▼
[T1.a 결정론] structure / self_correct / uncertainty   ← 0ms (즉시)
   │
   ▼
[T1.b LLM]    complexity 추정    (Haiku, 200 tokens)   ← 0.4초
   │
   ▼
[T1.c]       SeedVector blend(instant 0.4 + history 0.6)
   │
   ▼
[Mode]       select_mode(SeedVector) → Low/Mid/High
   │           ├─ Aporia 결정론 트리거 충족? → mode = "Nutrient"
   │           └─ Distress 결정론 트리거 충족? → mode = "Nutrient"
   ▼
[T2 LLM]     Question Designer 호출  (Sonnet, 600 tokens) ← 1.5초
   │           - input: text + mode + 헌법 분리편(현재 블록)
   │           - output: SocraticUtterance (JSON, 의문문 강제)
   │
   ▼
[Guardrail]  is_interrogative? → No → 1회 재생성 → 그래도 No → 정적 폴백
   │
   ▼
   학생 화면 (말풍선)                                   ← 누적 ≤ 2초
💡 지연 예산 5초 내: T1.b 0.4s + T2 1.5s + 네트워크 왕복 0.6s + UI 렌더 0.2s = 2.7초 평균. 5초 초과 시 "🧐 소크라테스가 골똘히 생각 중..." 진행 상태 노출.

제5조 (결정론 vs LLM — 측정 책임 분리)

한 신호의 측정 책임이 결정론과 LLM 사이에서 모호해지는 것을 방지하기 위해, 다음 분리표를 시스템 차원에서 강제한다.

측정 대상측정 방식구현
S_seed.structure결정론헌법 신호 매칭률 (B-TOPIC-01, B-ARG-METHOD 등 정규/구조 매칭)
S_seed.self_correct결정론이전 N개 블록과의 diff 점수
S_seed.uncertainty결정론"~인 것 같다 / ~일 수도 / 모르겠다" 정규식 매칭 빈도
S_seed.complexityLLMT1.b — Haiku 단일 호출 (0~1 스칼라만 반환)
Aporia 트리거결정론같은 주장 3회 + 응답시간 2σ + 부정 마커 — 3개 중 2개 충족
Distress 트리거결정론타이머 5분 미만 + 글자수 정체 + 부정 정서 + 항소 연속 — 휴리스틱 결합
Question Designer 발화LLMT2 — Sonnet, 가드레일·스키마 강제
Appeal 인정/기각LLMT5 — Sonnet, 독립 컨텍스트, rubric 강제
학생 5축 평가 (윤리성/반대의견 등)미구현 (P2)본 프로토타입 범위 외. 인터뷰 시 시연용 정적 결과만 표시.
📚 왜 결정론을 우선하는가? 인터뷰 평가에서 교사가 "왜 이때 Nutrient Mode로 전환됐냐?"고 물었을 때, "LLM이 그렇게 판단했다"는 답은 학술적으로 약하다. 결정론은 명시된 임계값으로 답할 수 있다 — Eisenberg et al. (2010)의 정서 조절 미성숙 개념과도 더 직접 매핑된다.

제6조 (가드레일 — Question Designer 강제 차단)

논문 §2.3은 시스템이 "blocks direct-answer outputs and instead routes all learner inputs through a Question Designer tool"라고 명시한다. 이는 본 시스템의 핵심 정체성이며, 위반 시 시스템 전체의 학술적 주장이 깨진다.

3중 가드레일

  1. 시스템 프롬프트 레벨 — "평서문, 답변, 정답 제시는 출력 자체가 차단된다" 명시
  2. 출력 스키마 레벨 — is_interrogative: bool 필드 필수, 의문부호(?)로 종결 검증
  3. 런타임 검증 레벨 — 응답 수신 후 정규식 /[?？][\s"']*$/ 매칭. 실패 시 1회 재생성, 그래도 실패 시 정적 풀에서 SOCRATES_LINES 폴백
⚠️ 운영 알람: 가드레일 폴백이 24시간 내 5회 이상 발생하면 시스템 프롬프트 회귀 가능성. 즉시 프롬프트 점검 트리거.

제7조 (폴백 — LLM 실패 / 지연 / 스키마 위반)

실패 유형대응
네트워크 실패 / 타임아웃 (≥5초) SOCRATES_LINES 정적 풀에서 (현재 모드 + 블록)에 해당하는 표준 대사 1개 무작위 선택. 차감된 잔여 횟수 복구. 토스트: "잠깐, 다시 한번 해볼래?"
JSON 스키마 위반 1회 재생성 (temperature 0.3 → 0.7). 2회 위반 시 정적 폴백.
Question Designer 가드레일 위반 (의문문 아님) 1회 재생성 with 강조 프롬프트 ("반드시 ?로 끝낼 것"). 2회 위반 시 정적 풀.
Appeal Judge 카테고리가 none이지만 upheld=true 스키마 모순 — upheld=false로 강제 보정 + 로그 알람
Complexity 추정 실패 (NaN / 범위 외) history 평균값으로 대체 (없으면 0.5 중간값). 측정 신뢰도 플래그 표시.
🎯 폴백의 목적: LLM 실패가 학생 경험을 깨뜨리지 않도록 한다. 학생은 "AI가 죽었다"는 느낌을 받지 않고, 표준 대사를 통해 마찰의 일관성을 유지한다.

제9조 (자기참조 편향 회피 — Appeal Judge)

Appeal Judge가 직전 비판을 자기가 판정하면 보수적 편향(자기 비판을 정당화)이 발생한다. 다음 절차로 회피한다.

  1. Question Designer가 생성한 비판은 별도 컨텍스트에 저장되며, Appeal Judge 호출 시 그 컨텍스트는 전달되지 않는다.
  2. Appeal Judge에게는 비판 텍스트와 학생 반박 텍스트만 전달된다 (출처 미공개).
  3. 판정은 rubric 3종(new_evidence / logical_flaw / perspective_shift)에 강제 매핑된다 — 자유 판단 금지.
  4. 인정/기각 통계가 한 세션에서 95% 이상 한쪽으로 치우치면 알람 (rubric 강도 점검 트리거).
📚 학술 근거: Bandura (2001) 자기효능감 이론에서 "정교한 상대를 이긴 정서적 체험"이 핵심이다. 만약 판정이 보수적이면 학생이 결코 이길 수 없게 되어 자기효능감 형성이 차단된다. 따라서 자기참조 편향 회피는 운영적 최적화가 아니라 학술적 정합성 확보의 영역이다.

제10조 (테스트 가능성)

결정론과 LLM의 분리는 테스트 전략에도 직접 반영된다.

대상테스트 방식설명

제11조 (학술적 정당성 매핑)

본 정책서의 모든 결정은 다음 인용에 매핑된다. 매핑 안 된 결정은 추가 정당성 없이 도입하지 않는다.

  • 최소 호출 / 결정론 우선 — Eisenberg et al. (2010), de Bruin et al. S2D2 (2023)
  • Question Designer 가드레일 — 논문 §2.3, Hmelo-Silver IBL (2007), TeachThought QBL (2022)
  • Appeal Judge 독립 컨텍스트 — Bandura (2001), Zenko Red Team (2015)
  • Aporia 결정론 트리거 — Vlastos (1991), Piaget (1985)
  • Distress 결정론 트리거 — Eisenberg et al. (2010), Ryan & Deci SDT (2000)
  • 모드 선택 — Bjork (1994), Vygotsky (1978), Kapur (2016)
  • Nutrient Mode — Novak et al. JiTT (1999), Csikszentmihalyi Flow (1990)
  • 페르소나 톤 — Dweck Growth Mindset (2006)

제12조 (변경 절차)

본 정책서의 변경은 다음 절차를 따른다.

  1. 새 LLM 호출 지점을 추가하려면, 학술적 인용을 명시하고 본 정책서의 제2조 표에 행을 추가한다.
  2. 새 가드레일을 도입하려면, 제6조에 4번째 가드레일 절차를 추가한다.
  3. 모델/예산 변경은 제8조 표를 갱신하고 SDS 「📋 버전 관리」 탭에 등재한다.
  4. 본 정책서가 충돌하는 새 도구를 제안받으면, 제2조 "5개 도구 동결" 원칙에 따라 거부가 기본값이며, 거부를 뒤집으려면 논문/이론 인용이 필요하다.
